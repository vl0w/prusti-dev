Expand description

Visit the reachable blocks of a CFG with a forward pass. During the visit, statements can be modified and injected. However, the structure of the CFG can not change. For each branch a context is updated, duplicated at forks, and merged with other contexts at joins.

Replace some statements, mutating the branch context

label is the label to be used for references when emitting fold/unfold operations. This is a hacky work-around for Silicon not being able to see that two locations are the same inside the package statement. The minimal example that illustrates the issue:

        None => list,
    }
